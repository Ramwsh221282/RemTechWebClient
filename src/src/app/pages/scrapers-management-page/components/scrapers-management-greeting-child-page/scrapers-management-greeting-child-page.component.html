Парсеры техники

Описание

Поддерживаемые сервисы парсинга:

AVITO.RU
DROM.RU

Для подробной информации о функционале сервисов парсинга разверните интересующую панель.

Описание
  • Сервисы парсинга - отдельные приложения, каждое из которых парсит назначенный ему сервис.
  • За управление сервисами парсинга отвечают объекты "Парсеры" которыми Вы можете управлять.
  • На каждый сервис парсинга существует свой единственный объект "Парсер" с уникальными названиями и типом парсинга (техника или запчасти).
  • Интеграция нового сервиса парсера осуществляется разработчиком путем создания нового небольшого приложения, предназначенного для парсинга сервиса.

Каждый сервис парсера работает автономно и ждёт своего времени запуска. Отключение основого приложения прерывает работу всех парсеров.

Объект "Парсер"
  • Объект парсер представляет собой единицу сервиса парсинга.
  • Объект парсер работает в трёх состояниях: "Ожидает", "Отключен", "Работает".
  • В каждой итерации парсера (в каждом моменте запуска парсера согласно расписанию) за процесс его работы может менятся его статистика времени выполнения и число добавленных объявлений.

    • В состоянии ожидания, парсер ждёт следующего запуска, согласно свойствам "Дни ожидания", "Следующий запуск".

    В состоянии "Отключен" парсер не будет запускаться даже если его "Следующий запуск" является релевантным по отношению к настоящему времени.

    В состоянии "Работает" парсер обрабатывает принадлежащие ему ссылки, создает "Журналы парсинга", в которых можно просмотреть, что делал парсер, а так же отправляет результаты собранных объявлений в основное приложение для их сохранения.

Объект "Ссылка парсера"
  • Ссылки парсера принадлежат парсеру, для которого они создаются.
  • На каждую ссылку парсера так же присутствует статистика, как и у парсера.
  • Для удобной работы с ссылками, рекомендуется давать им осознанное имя, например "Avito JCB".
  • Ссылки можно "Деактивировать", при таком случае, деактивированные ссылки будут игнорироваться при итерации парсинга.

Парсер не может хранить ссылки, которые дублируются по наименованию, либо по URL значению.

Парсер, у которого нет ссылок, нельзя перевести в состояние "Ожидает".

Если у ссылки и у парсера разный домен, операция добавления будет невалидной. Например нельзя для avito парсера добавить ссылку drom.ru/bulls.
